<p>
  Я провёл анализ требований для модуля "Регистрация и авторизация" <a href="https://intern.demoshopping.ru/">интернет-магазина</a>. Целью работы было выявление проблем в требованиях ещё до этапа разработки: неточностей, противоречий и недостающей информации.
</p>

<p>
  На основе анализа я подготовил различные виды тестовой документации:
</p>

<ul>
  <li>Чек-лист – проверки для основных пользовательских сценариев</li>
  <li>Тест-кейсы – детализированные сценарии с ожидаемыми результатами</li>
  <li>Баг-репорты – примеры оформленных дефектов</li>
</ul>
<p>
  Я протестировал мобильное приложение для Android<strong>«Список покупок»</strong> (Android). 
  Использовал реальное устройство и эмулятор Android Studio.
</p>

<ul>
  <li>Чек-лист – общие и функциональные проверки приложения.</li>
  <li>Тест-кейсы – проверка основных сценариев работы приложения</li>
  <li>Баг-репорты – найденные дефекты с описанием и шагами воспроизведения.</li>
</ul>
